The Basics

  • Signal - a value that can tell Angular when it changes
  • Computed - derive new value when one of the dependant signals change
  • Effect - a side-effectful operation which reads the value of zero or more signals. Automatically scheduled to be re-run whenever any of those signals is notified of a change.
More Info
  • Change detection only runs when the signal tells Angular it has changed.
  • Signals are small: 2kb.
  • Signals are fast.
  • The Angular team is working on interoperablity with existing state management libraries.
  • The plan is for Signals to eventually replace Zone.js, and for Signal-related lifecycle events.
